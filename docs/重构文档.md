# 智能简历优化系统 SaaS 平台重构架构文档

> **当前状态**: 已完成 P0-P3 本地化单节点开发，具备多 Agent 协同、SSE 实时流与 PDF/Word 本地解析导出能力。
> **重构目标**: 演进为高可用、多租户、可商业化变现的分布式 SaaS 平台，并为后续更广阔的求职生态打下数据基石。

---

## 一、 系统架构演进 (SaaS 底座建设)

为了应对多用户并发，系统必须从单机内存驱动转变为真正的分布式架构。

### 1.1 任务调度与执行解耦
* **现状**: 当前依赖 FastAPI 的 `BackgroundTasks` 在主进程的后台执行大模型流，并通过 `task_store.py` 的 `asyncio.Queue` 维护内存级别的任务状态。
* **重构动作**:
    * 引入 **Celery + Redis** 任务队列体系。
    * 将核心的 `run_optimize_task` 封装为独立的异步 Worker。
    * **收益**: 防止长耗时的 Agent 链式调用拖垮 API 网关，支持计算节点的水平动态扩容。

### 1.2 状态持久化与实时通信剥离
* **现状**: LangGraph 节点数据总线依赖内存中的 `ResumeGraphState`，且 SSE 事件流直接读取内存投递。
* **重构动作**:
    * **通信层**: 采用 **Redis Pub/Sub**。Worker 节点执行时将 `node_start`、`review_feedback` 等事件推入 Redis，FastAPI 仅作为订阅者通过 SSE 转发给前端，实现完全解耦。
    * **状态层**: 为 LangGraph 引入 `AsyncPostgresSaver` 检查点机制。确保工作流由于外部原因（如 LLM API 超时）中断时，能从断点（如 Node 2 JD 分析师）无缝恢复。

## 二、 核心数据模型与资产沉淀

SaaS 平台的核心护城河在于用户资产的管理与复用。引入 **PostgreSQL** 替代现有的内存管理。

### 2.1 数据库核心表结构 (ER 设计)

| 实体模型 | 核心字段与说明 | 业务价值 |
| :--- | :--- | :--- |
| **Users** | `user_id`, `auth_provider`, `credits`, `subscription_tier` | 支撑多租户隔离与计费体系。 |
| **Tasks** | 替代目前的 `TaskRecord`。关联 `user_id`，记录 `status`, `target_jd_text`, `duration_ms`。 | 任务历史追踪与系统性能审计。 |
| **Resumes** | 存储由 Node 5 (终审排版员) 产出的 `RenderReadyResume` 完整 JSON 数据。 | 支持前端随时拉取历史版本，重新利用 `@media print` 渲染或导出 Word。 |
| **UserProfiles** | 结构化能力画像池，包含 `skill_matrix`, `raw_experiences` 等。 | **核心资产池**。将 Node 1 提取的客观事实独立建表，构建持续的个人能力档案。这使得底层数据可以无缝流入 TechSpar 等 AI 面试辅导系统，彻底打通“简历优化 → 定制化面试对练”的数据闭环。 |

## 三、 商业化变现引擎 (Auth & Billing)

### 3.1 身份认证中心
* 首轮重构以 **邮箱注册 / 邮箱密码登录 + JWT** 为最小可交付闭环，OAuth 2.0（Google/GitHub/微信）保留为后续增量能力。
* 使用 FastAPI 的 `Depends` 依赖注入机制，全面接管并保护现有的 `/api/v1/optimize`、`/api/v1/export/docx/{task_id}` 等核心交互路由。
* 前端统一通过 `Authorization: Bearer <token>` 携带访问令牌，未认证请求由后端稳定返回 401。

### 3.2 计费与额度模块
* **Token/信用点体系**: 
    * 每次成功流转完整的 5 节点图编译运行（从 `profile_builder` 到 `final_typesetter`）扣除固定额度。
    * 如果触发了 Node 4 (内容审查员) 的打回重试（上限 3 次），作为平台成本由系统承担，不额外向用户扣费，以保证用户体验。
    * 如果任务在正式进入工作流前因参数错误、文件解析失败或平台异常中断，首版不扣费，优先保证计费可信度。
* **SaaS 订阅分层 (Tiers)**:
    * *Free Tier*: 基础排版、限制月度优化次数、携带平台水印。
    * *Pro Tier*: 解锁无限次优化、高级版 PDF 模板导出、永久历史版本云端保留。

### 3.3 加入后台管理功能
* 首版仅提供最小化 Admin API：用户列表、任务审计、基础统计、人工额度调整。
* 复杂后台界面、支付对账与细粒度 RBAC 不作为 R3 首轮范围，避免阻塞认证与计费主链路。

## 四、 客户端体验

### 4.1 Web 端控制台演进
在 P2 阶段打造的 Vue 3 + Vite "Modern Premium" 视觉标准上，新增以下模块：
* **Dashboard 工作台**: 统一展示额度消耗图表、历史优化任务流以及用户当前的底层能力画像池。
* **主题集市**: 将目前的内联 CSS 变量 解耦封装，提供多套可切换的渲染主题（如“互联网极简”、“金融严谨风”）。

## 五、 LLM 治理与降本增效

### 5.1 智能路由与容灾备份
* 优化 `services/llm_service.py`，建立动态 Fallback 机制：若首选的高成本模型（如 GPT-4o）触发并发限流，自动无缝降级至 Claude-3.5-Sonnet 或 Gemini 1.5 Pro，保障 SaaS 的高可用性 (SLA)。

### 5.2 审计与成本归因
* 在工作流记录中，精确统计每次图节点流转所消耗的 Prompt Token 与 Completion Token。将模型开销与具体的 `task_id` 和 `user_id` 绑定，用于计算精准的单个用户获取成本 (CAC) 与服务成本 (COGS)。
