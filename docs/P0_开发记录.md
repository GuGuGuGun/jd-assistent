## 三、核心设计决策

### 3.1 三重防线 LLM 输出机制

所有结构化输出均通过 `safe_llm_generate()` 生成：

1. **API 级防线**：`response_format={"type": "json_object"}`，不支持时自动回退
2. **物理级防线**：正则提取 JSON，兼容 Markdown 包裹和裸 JSON
3. **代码级防线**：`json.loads` + `Pydantic.model_validate`，失败后将错误信息回传给 LLM 重试（最多 3 次）

### 3.2 工作流编排

```
profile_builder → jd_analyst → content_optimizer → content_reviewer
                                       ▲                    │
                                       └────── 打回 ────────┘
                                                             │ 通过
                                                     final_typesetter → END
```

- Node 1 与 Node 2 无数据依赖，采用顺序执行，效果等价于并行
- Node 4 审查未通过时循环回 Node 3，最多重试 3 次
- 超过重试上限后强制进入 Node 5 排版

### 3.3 Prompt 管理策略

- Prompt 模板独立存放于 `prompts/` 目录，与代码解耦
- 使用 `{variable}` 占位符，运行时通过 `str.format()` 注入
- 每个 Prompt 遵循"角色锚定 → 硬性约束 → 输入 → 输出格式"四段式结构
