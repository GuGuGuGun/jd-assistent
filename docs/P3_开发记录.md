## 二、架构设计与特性实现

### 2.1 后端文件提取引擎 (`parser.py`)
为了取代原有的“纯文本复制粘贴”，实现了健壮的简历文档解析中间件。
- **PDF 解析**：集成了 `pdfplumber`，相比其它库能更准确地提取带有复杂排版和表格的简历文本流。
- **Word 解析**：集成了 `python-docx`，安全抽取 `.docx` 的全部段落内容。针对遗留的 `.doc` 做了容错处理及降级提示。
- **纯文本/Markdown 解析**：原生支持 `utf-8` 解码，并设置了 `gbk` 容错处理。

### 2.2 API 端点升级 (`routes.py`)
安全地将 `/api/v1/optimize` 接口从接收 `application/json` 重构成了接收 `multipart/form-data`。
- 入参使用了 FastAPI 的 `UploadFile = File(...)` 及 `jd_text = Form(...)`。
- 在将其送入后台 `run_optimize_task` 大模型流之前，前置同步提取 `resume_text` 纯文本，若文件内容损坏则直接返回 `HTTP 400`。

### 2.3 现代化“拖放式”上传组件 (`ResumeUploader.vue`)
废弃了左侧的普通 `textarea`，升级成了符合 “Premium” 高级视觉标准的拖放拦截区域：
- **微交互**：支持原生的 Drag & Drop (拖拽) 事件监听 (`@dragover`, `@drop`)，拖入时虚线框变为蓝色实体框并行发光 (`shadow-[0_0_0_4px_rgba(59,130,246,0.1)]`)。
- **动态回显**：用户上传或选中文件后，实时渲染一个漂亮的文件名回显卡片并自动计算格式化后的文件大小（KB/MB）。同时提供红色的“移除重选”按钮增加容错度。
- **底层封装**：改写了 `axios` 提交逻辑，自动构建 `FormData` 装载 `resume_file` 二进制流发送至后端。

### 2.4 Word (DOCX) 文档无缝导出
为了打通简历优化的闭环体验，除了浏览器原生的“纯净版 PDF 打印”功能外，增加了原生 Word 导出。
- **后端生成器 (`docx_exporter.py`)**：利用 `python-docx` 遍历任务对象中的 `RenderReadyResume` 数据流，硬编码了字号、中文字体 (微软雅黑)、加粗与段落缩进等排版要素，重组出立等可用的 MS Word 实体文档。
- **API 下载**：新增 `GET /api/v1/export/docx/{task_id}` 端点，通过 `StreamingResponse` 返流二进制，自动挂载 UTF-8 编码的文件名，从而被浏览器无缝识别为下载任务。
- **前端控件**：在任务进度完成的最后视图中，增加“导出 Word”功能按钮与 PDF 导出并列，允许用户一键获取所有结果继续编辑。

3. **额度与后台管理**

### 4.3 本轮加固结果

针对首版 R3，又补了一轮边界修正，避免把 MVP 做成“能跑但不稳”的状态：

- **SSE 查询令牌收口**：`access_token` 查询参数现在只允许用于 SSE 路由，普通受保护接口统一只认 Bearer Token。
- **扣费并发安全**：补充任务级扣费审计字段，确保同一任务即使并发触发完成回调，也最多只扣 1 次额度。
- **前端额度同步**：任务完成后的最终结果拉取，会顺带刷新 `/auth/me`，头部额度不再滞后。
