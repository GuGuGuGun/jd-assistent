## 二、核心设计说明

### 2.1 任务生命周期管理 (`task_store.py`)

- **数据结构**: 
  - `TaskRecord`: 记录任务整体状态 (`processing`, `completed`, `failed`)
  - `NodeLogEntry`: 记录单个节点状态 (`pending`, `running`, `done`, `error`)、耗时
- **实时通信**: 
  - 每个任务维护独立的 `asyncio.Queue` 用于发送 Server-Sent Events (SSE)
  - 通过单例 `task_store` 全局管理任务数据

### 2.2 异步工作流执行 (`resume_service.py`)

- 将 LangGraph 工作流封装为 `run_optimize_task` 函数
- 通过 FastAPI 的 `BackgroundTasks` 将优化任务推入后台执行，立即响应客户端任务 ID
- 使用 LangGraph 的 `astream()` 方法监听执行进度：
  - 每触发一个节点，即向 `task_store` 记录 node_start / node_complete
  - 特殊捕获 Node 4 (审查员) 的反馈结果，并推送为独立事件
  - 提取 Node 5 (排版员) 生成的关键 JSON，保存至 `TaskRecord.result` 并标记任务完成

### 2.3 SSE 事件流 (`api/routes.py`)

实现 `GET /api/v1/tasks/{task_id}/stream`，支持以下事件：

| 事件类型 | Data 格式示例 | 触发时机 |
|----------|---------------|----------|
| `node_start` | `{"node": "profile_builder", "message": "正在提取..."}` | 节点开始执行 |
| `node_complete`| `{"node": "profile_builder", "duration_ms": 3200}` | 节点执行完成 |
| `review_feedback`| `{"node": "content_reviewer", "passed": false, "feedback": "..."}` | 审查员输出结果 |
| `complete` | `{"task_id": "xxx", "message": "完成"}` | 整个工作流结束 |
| `error` | `{"error": "错误信息"}` | 发生异常 / 任务失败 |
| `heartbeat`| `{"ping": true}` | 无新事件时每 30s 发送，防连接断开 |

### 2.4 全局异常防御 (`main.py`)

- **ValueError**: HTTP 400 (请求参数 / 业务校验错误)
- **RuntimeError**: HTTP 500 (LLM 调用失败 / 重试超限)
- **Exception**: HTTP 500 (兜底未知错误)
- **CORS**: 允许所有跨域请求，方便后续前端本地联调 (`allow_origins=["*"]`)
